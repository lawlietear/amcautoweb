## [2.0.0] - 2026-03-12

### 🎯 版本概述
重大功能升级版本，支持多用户并发访问，新增收购业务模板，全面增强隐私保护和用户体验。

### ✨ 新增功能
- **多用户并发支持** - 升级为 Waitress 生产服务器，支持 8 线程并发，适合团队协作
- **新增收购业务模板** - 风险合规部新增"收购业务批复"模板（含 14 个变量）
- **自定义日期筛选** - 操作日志支持指定开始/结束日期的精确筛选
- **隐私保护模式** - 最近使用模块脱敏显示，隐藏债务人名称、合同编号等敏感信息
- **自动 IP 显示** - 启动时自动获取并显示本机局域网 IP 地址
- **示例数据完善** - 收购业务模板添加完整的 sample_data 示例

### 🔧 功能优化
- **业务部字段调整** - 批复信息标记为"必填，否则无法生成合同"
- **统计逻辑优化** - 统计范围改为全部字段（必填/非必填分别显示）
- **字体大小调整** - 首页"山东金融资产"24px，部门名称 30px
- **图标风格统一** - 所有 emoji 图标替换为 Lucide SVG 线条风格
- **模板路径优化** - 配置加载时自动将相对路径转换为绝对路径

### 🐛 问题修复
- **修复生成失败问题** - 解决 FormatUtils 变量作用域问题导致的风险合规部生成失败
- **修复文件名中文括号** - 批复编号中的中文括号正确处理
- **修复缓存防重逻辑** - 适配风险合规部字段（approval_no + project_name）
- **修复今日生成统计** - 生成成功后正确更新今日生成数量
- **修复静态文件缓存** - 添加版本号避免 CSS/JS 缓存问题

### 📚 文档
- **DEPLOY_GUIDE.md** - 新增完整部署指南（含环境配置、防火墙设置、启动说明）
- **requirements.txt** - 新增 Python 依赖清单
- **CHANGELOG.md** - 新增版本变更日志

### 🔒 安全性
- **敏感信息脱敏** - 最近使用不再存储文件名、债务人名称、编号等
- **只显示部门+类型** - 改为显示"业务部 - 合同文件"等脱敏信息

### 🚀 部署信息

#### 环境要求
- **操作系统**：Windows 10/11
- **Python**：3.8 或更高版本
- **网络**：局域网内可互通

#### 快速部署
```bash
# 1. 安装依赖
pip install flask docxtpl pandas openpyxl waitress

# 2. 配置防火墙（PowerShell 管理员）
netsh advfirewall firewall add rule name="AMC文书系统" dir=in action=allow protocol=tcp localport=5000

# 3. 启动服务
python app.py
```

#### 技术参数
| 配置项 | 值 | 说明 |
|-------|-----|------|
| 服务器 | Waitress | 支持多人并发 |
| 端口 | 5000 | 可修改 |
| 线程数 | 8 | 支持 8 个并发请求 |
| 模板路径 | 自动适配 | 相对路径转绝对路径 |
| 输出目录 | generated/ | 自动创建 |

## [2.1.0] - 2026-04-02

### 🎯 版本概述
代码质量优化版本，重构核心逻辑，提升可维护性和性能，无功能变更。

### 🔧 代码重构
- **配置集中化** - 新建 `core/config.py`，所有硬编码配置迁移至统一模块
- **函数拆分** - `api_generate()` 从 289 行拆分为 5 个辅助函数，主函数降至 80 行
- **路由合并** - `risk_compliance()` 和 `business()` 合并为统一的 `dept_page()`，保留向后兼容
- **日志系统** - 全量替换 `print` 为 `logging`，新增 `RotatingFileHandler` 日志轮转

### ⚡ 性能优化
- **Excel增量更新** - `update_status()` 改用 `openpyxl` 直接操作单元格，避免全量读写
- **文件名生成** - 简化 `_generate_filename()`，使用 `str.format()` 替代手动替换
- **重复import清理** - 删除函数内的重复 `import`，统一移至文件顶部

### 📊 代码质量
| 指标 | 修改前 | 修改后 |
|------|--------|--------|
| app.py 总行数 | 719 行 | 674 行 |
| api_generate 函数 | 289 行 | ~80 行 |
| print 语句 | 20+ 处 | 0 处 |
| 硬编码配置 | 多处 | 全部集中 |

### 🔒 兼容性
- 所有 API 端点行为保持不变
- 旧路由 `/business` 和 `/risk_compliance` 完全兼容
- 模板渲染逻辑无变更
- 文件生成格式无变更

## [2.1.2] - 2026-04-04

### 🐛 问题修复
- **必填字段高亮彻底修复** - 解决 Handsontable 虚拟渲染导致的高亮状态错乱问题
  - 原因：`afterChange` 回调在虚拟滚动时无法正确追踪所有单元格状态
  - 方案：改用 `afterRenderer` 回调，在每次单元格渲染时动态添加/移除 CSS 类
  - 实现：`required-empty`（红色底）和 `required-filled`（绿色底）根据当前值实时切换

### ✨ 新增功能
- **企业微信智能表格集成** - 风险合规部新增快捷入口按钮
  - 一键打开企业微信智能表格，方便复制批复信息
  - 按钮位置：部门页面右上角，与业务部保持一致
  - 自动提示用户复制信息后返回粘贴
